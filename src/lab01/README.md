# Реализованный класс: Apartment

### Краткое описание процесса
Класс — квартира (Apartment). Характеристики объекта недвижимости: адрес, площадь, количество комнат, цена, этаж и год постройки. Добавлена инкапсуляция и валидация.
Логика проверок вынесена в отдельный модуль validate.py для удобства тестирования и возможного переиспользования в других классах.

Также добавлены бизнес-методы, отражающие реальные операции с недвижимостью: продажа, бронирование, расчет ипотеки. Введены состояния квартиры (доступна, забронирована, продана и т.д.).

Реализованы магические методы для удобной работы с объектом и создан демонстрационный файл, показывающий все возможности класса.

### Краткое описание класса

Apartment — это класс, представляющий квартиру в системе недвижимости. 

Содержит поля:
__address - адрес квартиры
__area - площадь в квадратных метрах
__rooms - количество комнат
__price - цена в рублях
__floor - этаж
__construction_year - год постройки
__status - состояние квартиры
__reserved_until - дата окончания брони

# Методы класса
- `can_be_sold` — Проверяет, можно ли продать квартиру
- `sell` — 	Выполняет продажу квартиры
- `reserve(days=7)` — Бронирует квартиру на указанное количество дней
- `set_unavailable()` — Переводит квартиру в статус "на ремонте"
- `calculate_monthly_payment(years, percent)` — Рассчитывает ежемесячный платеж по ипотеке

**Магические методы:**
- `__init__` — Конструктор
- `__str__` — Для пользователей
- `__repr__` — Для разработчиков
- `__eq__` — Сравнение на равенство

### Модуль validate.py

Все функции валидации вынесены в отдельный модуль для простоты использования:

- `validate_address()` — Проверка адреса
- `validate_area()` — Проверка площади
- `validate_rooms()` — Проверка количества комнат
- `validate_price()` — Проверка цены
- `validate_floor()` — Проверка этажа
- `validate_construction_year()` — Проверка года постройки

Каждая функция при успешной проверке возвращает очищенное значение, при ошибке выбрасывает исключение TypeError или ValueError.
